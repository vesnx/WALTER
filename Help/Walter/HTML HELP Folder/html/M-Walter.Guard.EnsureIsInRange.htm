Remarks

This method validates that value falls within the inclusive range defined by min and max. If the value is outside of this range, an [ArgumentOutOfRangeException] is thrown, detailing the expected range and the actual value. Additionally, the error is automatically logged using an [ILogger] instance configured in Walter.Guard, if available. If an [ILogger] is not configured, the exception details are intended to be logged to the operating system's event log, ensuring that the error is recorded for diagnostic purposes.

Examples

The following example demonstrates how to use EnsureIsInRange to validate that a method parameter is within an acceptable range:
C#
public void SetTemperature(int temperature)
{
    Guard.EnsureIsInRange(temperature, 0, 100);
    // Proceed with setting the temperature as it is within the acceptable range.
}
 This validation ensures that the 'temperature' parameter is between 0 and 100, inclusive, throwing an informative exception if not.
